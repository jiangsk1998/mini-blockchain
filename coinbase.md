# Coinbase 交易（挖矿奖励）

## 先搞懂：普通交易 vs Coinbase 交易

**普通交易**：Alice 有钱 → 转给 Bob（钱从 Alice 账户扣，总量不变）

**Coinbase 交易**：系统凭空 → 发给矿工（没有发送方，总量增加）

Coinbase 是区块链**增发新币**的唯一合法渠道。矿工打包区块、完成挖矿，系统奖励一笔新币作为报酬，这就是比特币等加密货币的货币发行方式。

---

## 为什么叫"Coinbase"？

Coinbase = Coin（硬币）+ Base（基础/来源），意思是"货币的来源"，即新币诞生的地方。

> 著名交易所 Coinbase 就是取自这个词，意为"加密货币的入口"。

---

## 和手续费的区别

矿工的收入来源有两块，容易混淆：

| 来源 | 是什么 | 钱从哪来 |
|------|--------|----------|
| 手续费 | 用户支付给矿工的服务费 | 从转账方账户扣 |
| Coinbase 奖励 | 系统发给矿工的挖矿奖励 | 凭空新增，不扣任何人 |

比特币早期每块奖励 50 BTC（全是新币），手续费微乎其微。随着减半，奖励越来越少，手续费占比越来越大。

---

## 比特币的减半机制

比特币每挖出 21 万个区块（约 4 年），奖励减半一次：

```
第 0 ~ 21万块：每块奖励 50 BTC
第 21万 ~ 42万块：每块奖励 25 BTC
第 42万 ~ 63万块：每块奖励 12.5 BTC
...直到约 2140 年奖励趋近于 0
```

总供应量上限：**2100 万 BTC**（由数学公式保证，不会超发）。

## 在代码里怎么实现

**核心思路：Coinbase 交易是一笔特殊交易，没有发送方**

```rust
// 普通交易：from = Alice的地址
// Coinbase：from = "coinbase"（特殊标记，不是真实地址）

Transaction {
    from: "coinbase".to_string(),  // 标记为系统增发
    to: miner_addr,                // 矿工地址
    amount: BLOCK_REWARD,          // 固定奖励金额
    fee: 0,                        // Coinbase 不收手续费
    ...
}
```

**挖矿时的处理流程：**

```
mine_pending_transactions():
  1. 构造 Coinbase 交易（from="coinbase", to=矿工, amount=奖励）
  2. 把 Coinbase 放到交易列表第一位
  3. 后面跟普通交易
  4. 打包区块、挖矿
  5. 执行交易时：
     - 遇到 from="coinbase" → 直接给矿工加钱（不扣任何人）
     - 遇到普通交易 → 正常转账逻辑
```

**跳过验签：**

Coinbase 没有真实发送方，不需要私钥签名，`verify_signature` 里需要特判：

```rust
if self.from == "coinbase" {
    return true;  // Coinbase 直接通过，不验签
}
```

**跳过余额检查：**

同理，`add_transaction` 不需要对 Coinbase 检查余额（系统增发，无需余额）。Coinbase 交易由矿工在挖矿时自动生成，不走 `add_transaction` 流程。

---

## 实现要点总结

| 问题 | 处理方式 |
|------|----------|
| 如何标识 Coinbase | `from = "coinbase"` |
| 钱从哪来 | 直接给矿工加余额，不扣任何账户 |
| 需要签名吗 | 不需要，跳过验签 |
| 需要余额检查吗 | 不需要，跳过 |
| 放在区块哪里 | 第一笔交易（比特币规范） |
| 奖励金额 | 固定常量，或按区块高度减半 |
